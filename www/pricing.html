Frequently asked

Straight answers about how Nivaro is licensed, deployed, and supported.

Yes. Nivaro is MIT licensed and the full source is on GitHub. There is no feature-gating, no usage limits, and no telemetry. Every feature — workflows, AI, multi-workspace, SLA tracking — ships in the open-source build. The only paid options are support contracts for teams that want hands-on help from the people who built it.
You run the Nivaro API and admin UI on your own servers or cloud account. A docker-compose.yml is included that brings up the API, admin, Redis, Inngest, and a database in one command. You own the runtime, the data, and the keys. Nothing phones home. You can deploy to any Docker-compatible environment — VMs, Kubernetes, bare metal, AWS ECS, Azure Container Apps, etc.
Absolutely. Community is a complete, production-ready deployment — not a trial. When your team decides they want faster support response or architectural guidance, just reach out. There's no migration involved; you're already running the same software. Pro support simply adds people and access, not a different build.
Nivaro was built specifically to replace Directus 11, which moved enterprise features behind a paid plan at v11.17+. The key differences: Nivaro targets MSSQL and fixed production schemas (no dynamic schema editor), has a built-in state machine workflow engine, a multi-dimensional pipeline/owner matrix, SLA tracking, business-hours alerts, and deep Microsoft OIDC + Entra integration — all in the open-source build with no cloud upsell. It's a narrower tool that goes much deeper on operational workflows.
Yes — Nivaro Cloud is the managed option. Same open-source software, fully hosted: provisioning, backups, DNS, and updates handled for you. Plans start free. If you'd rather run it yourself, the self-hosted route is always available and always free.
